# Unity_ClickerGame_StarLight

Unity로 제작한 방치형 클릭커 게임 프로젝트입니다.  
재화 획득, 업그레이드, UI 갱신, 사운드 관리, 데이터 저장 등 클릭커 게임의 핵심 시스템을 직접 구현했습니다.

> 현재 공개 버전은 광고 기능 및 인앱 결제 기능을 제거한 버전입니다.

---

## 프로젝트 소개

이 프로젝트는 Unity를 활용해 클릭커 게임의 핵심 구조를 직접 구현해보는 것을 목표로 제작했습니다.  
단순히 버튼을 누르는 기능만이 아니라, **유저 데이터 관리**, **업그레이드 시스템**, **UI 반영**, **리소스/사운드 관리** 등 실제 게임 클라이언트에서 자주 다루게 되는 요소들을 직접 구성했습니다.

프로젝트를 진행하며 기능을 빠르게 추가하는 것뿐 아니라,  
기능별로 폴더와 클래스를 나누고 관리 역할을 분리하여 **유지보수가 가능한 구조**를 만드는 것을 함께 고민했습니다.

---

## 개발 환경

- **Engine**: Unity 2022.3.8f1
- **Language**: C#
- **Platform**: Windows / Mobile 대응 고려
- **Version Control**: Git / GitHub

---

## 주요 기능

- 클릭을 통한 재화 획득
- 재화를 사용한 업그레이드 시스템
- 유저 데이터 저장 및 불러오기
- 사운드 설정 및 볼륨 관리
- 언어/튜토리얼 여부 등 유저 설정 데이터 관리
- UI 갱신 및 게임 진행 상태 반영
- 리소스(사운드, 스프라이트, 이펙트) 관리 구조 분리

---

## 프로젝트 구조

```text
Assets/Scripts
├─ Data
├─ Define
├─ Enum
└─ Monobehavior
   ├─ Controller
   ├─ Manager
   ├─ Pool
   ├─ ResourceManager
   └─ UI
```

기능별로 폴더를 분리하여 역할을 나누었고,  
특히 ResourceManager와 UI, 데이터 관련 클래스를 분리해 각 시스템의 책임을 명확하게 유지하려고 했습니다.

---

## 구현하면서 중점적으로 고민한 점

### 1. 데이터와 화면 표시를 분리하려고 노력
클릭커 게임은 수치가 자주 바뀌고, 그에 따라 UI도 계속 갱신됩니다.  
이 과정에서 데이터 로직과 화면 표시 로직이 강하게 섞이면 유지보수가 어려워지기 때문에,  
유저 데이터와 UI 관련 처리를 가능한 한 분리하는 방향으로 구조를 잡았습니다.

### 2. 리소스 관리 클래스 분리
사운드, 스프라이트, 이펙트처럼 성격이 다른 리소스를 한 곳에 몰아넣기보다  
각각 관리 클래스를 나누어 확장성과 가독성을 높이고자 했습니다.

### 3. 저장 데이터 구조화
재화, 업그레이드, 튜토리얼 여부, 언어 설정, 사운드 볼륨 등  
유저의 진행 상태를 하나의 데이터 구조로 관리해 저장/불러오기 흐름을 일관되게 유지하려고 했습니다.

## 배운 점

이 프로젝트를 통해 Unity에서 게임 기능을 구현하는 것뿐 아니라,  
**기능이 늘어날수록 클래스를 어떻게 나누고 어떤 책임을 맡길지** 고민하는 경험을 할 수 있었습니다.  
또한 클릭커 장르 특성상 반복적으로 갱신되는 데이터와 UI를 다루면서,  
게임 클라이언트 개발에서 중요한 **상태 관리와 구조 분리의 필요성**을 배웠습니다.
